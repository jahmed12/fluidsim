Navier stokes equation we need to use:

4 components to the  navier stokes equation for incompressible fluids:

  • Advection:
  • Movement of a fluid based on material properties and external forces on a fluid
  • Particle movement within the fluid is modeled using u (which is a vector field, specifically a velocity field).
  • ∇ is our vector of spatial partial derivatives

  • Diffusion:
  • V is the kinematic viscosity, for smoke (air) is 0.0171 η(mPa·s)
  • This describes the fluid's ability to spread out and dissipate.
  • External forces:
  • The f variable in this equation. For our simulation we plan on having a constant upwards buoyant force so our f will be non-zero
  • Pressure:
  • 𝜌 is the density of the fluid
  • p is the pressure

Each term needs to be calculated individually and shaders need to be applied accordingly. We then combine all 4 for the final shader result.

Progress made (physics -> code):

For actually implementing Navier Stokes as code, we plan to pair three.js with webGL to create and render our interactive simulation within a 2D grid in a browser. We will implement our code in SIMD fashion on a GPU to parallelize the computation of our fragments. This will speed up the simulation while preserving proper functionality. In order to obey the simulation’s physical constraints, we will use a vorticity confinement model, which will maintain proper locality of the fluid’s rotational and kinematic movement. That way, different fluids, especially those with low viscosity (V), will move more realistically based on their properties, like density and pressure . We will use the OpenGL Shading Language (GLSL) to build our shaders, which each play a particular role in each time step of our simulation. First, the advection shader will use the implicit Euler method to update some property of the fluid (density, velocity, concentration, temperature, etc.) at each grid position using the previous timestep’s velocity vector. The fields we want to update and the previous timestep’s velocity vector will be passed in as arguments to this shader. After, the diffusion shader will use the inputted viscosity argument to capture the fluid’s dissipation via the Jacobi iteration technique. Next, the vorticity and vorticityForce shaders will use adjacent fragment values to update the velocity vector to account for the fluid’s vorticity. Then, the projection shader updates the fluid’s pressure, which is solved using the Poisson-pressure equation, calculates its gradients, and then subtracts them from the velocity vector. Finally, we will enforce that the velocity field never leaves the display grid using proper boundary checking.
